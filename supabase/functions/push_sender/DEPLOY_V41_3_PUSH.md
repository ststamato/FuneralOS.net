# V41.3 — Push Notifications (ειδοποίηση με κλειστή εφαρμογή)

Στόχος: όταν ένας συνάδελφος αλλάζει κάτι, να έρχεται **ειδοποίηση στο κινητό
ακόμη κι αν η εφαρμογή είναι κλειστή**, όπως σε native iOS app.

## Τι προστέθηκε (μόνο προσθήκες — τίποτα δεν αλλάζει στον υπάρχοντα κώδικα)
1. `sw.js` (στη ρίζα, δίπλα στο index.html) — Service Worker που λαμβάνει το push
   και ανοίγει την εφαρμογή στο tap. **Δεν έχει fetch handler**, άρα δεν επηρεάζει
   φόρτωση/cache.
2. `supabase/functions/push_sender/index.ts` — το Edge Function που στέλνει τα push.
   Ο client ήδη το καλούσε· απλώς τώρα υπάρχει και δουλεύει.

## ⚠️ ΚΡΙΣΙΜΟ: όρια iOS (όχι περιορισμός δικός μας — της Apple)
- Δουλεύει σε **iOS/iPadOS 16.4 ή νεότερο**.
- Η εφαρμογή πρέπει να είναι **εγκατεστημένη** (Safari → Share → "Add to Home Screen").
  Σε απλό tab του Safari, η Apple ΔΕΝ επιτρέπει push με κλειστή εφαρμογή.
- Κάθε συνάδελφος πρέπει μία φορά να ανοίξει την εγκατεστημένη εφαρμογή και να
  πατήσει **"🔔 Push"** (στο modal Updates) για να δώσει άδεια.
- Χρειάζεται HTTPS (το Cloudflare το παρέχει αυτόματα).
- Σε Android/Chrome δουλεύει πιο χαλαρά (δεν απαιτείται εγκατάσταση).

## Βήματα ενεργοποίησης (μία φορά, από τον iMac)

### 1) VAPID κλειδιά
Το `app.js` έχει ήδη VAPID_PUBLIC_KEY. Χρειάζεσαι το **ιδιωτικό** κλειδί που ταιριάζει.

- **Αν το έχεις ήδη** (από τότε που φτιάχτηκε το public): προχώρα στο βήμα 2.
- **Αν ΔΕΝ το έχεις**: φτιάξε νέο ζευγάρι:
  ```bash
  npx web-push generate-vapid-keys
  ```
  Θα σου δώσει Public + Private. ΤΟΤΕ πρέπει να βάλεις το νέο **Public** στο `app.js`
  (γραμμή `const VAPID_PUBLIC_KEY = "..."`) και να ξανα-ανεβάσεις το app στο Cloudflare.
  (Πες μου και το αλλάζω εγώ στο αρχείο.)

### 2) Secrets στο Supabase (δεν μπαίνουν ΠΟΤΕ στον client)
```bash
supabase secrets set VAPID_PUBLIC_KEY=το_public_κλειδι
supabase secrets set VAPID_PRIVATE_KEY=το_private_κλειδι
supabase secrets set VAPID_SUBJECT=mailto:ststamato@gmail.com
```
(Το `SUPABASE_URL` και το `SUPABASE_SERVICE_ROLE_KEY` υπάρχουν ήδη αυτόματα.)

### 3) Deploy της συνάρτησης
```bash
supabase functions deploy push_sender
```

### 4) Ανέβασε το `sw.js` στο Cloudflare
Πρέπει να βρίσκεται στην **ίδια ρίζα** με το `index.html` (URL: `https://.../sw.js`).
Στο τελικό zip είναι ήδη στη σωστή θέση.

## Πώς δοκιμάζεται
1. Εγκατέστησε την εφαρμογή σε 2 κινητά (Add to Home Screen), άνοιξέ την, πάτησε "🔔 Push".
2. Κλείσε την εφαρμογή στο κινητό Α.
3. Από το κινητό Β άλλαξε/πρόσθεσε μια τελετή.
4. Σε λίγα δευτερόλεπτα το κινητό Α παίρνει ειδοποίηση. ✅

## Ασφάλεια
- Καμία αλλαγή σε λειτουργία της εφαρμογής. Αν δεν στηθούν τα κλειδιά, απλώς δεν
  έρχονται push — όλα τα υπόλοιπα δουλεύουν κανονικά (όπως σήμερα).
- Η εφαρμογή έχει ήδη και τοπικές ειδοποιήσεις (όταν είναι ανοιχτή) ως fallback.
