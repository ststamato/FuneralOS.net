V41.2 — Hermes AI Agent (Agent Tools / function-calling)

Τι προστέθηκε:
- Ο Cloud AI έγινε πραγματικός AGENT. Αντί να του στέλνουμε όλα τα δεδομένα ωμά,
  ο Ερμής έχει ΕΡΓΑΛΕΙΑ (read-only) και τα καλεί μόνος του όσες φορές χρειαστεί:
    1. search_ceremonies  — αναζήτηση τελετών (λέξεις-κλειδιά, χρόνος, ταφή/αποτέφρωση)
    2. count_ceremonies   — μέτρηση/ομαδοποίηση (υπεύθυνος, τόπος, ΣΕΤ, φέρετρο, κ.λπ.)
    3. warehouse_alerts   — ειδοποιήσεις χαμηλού αποθέματος
    4. list_notes         — σημαντικές σημειώσεις (μέγιστη προτεραιότητα)
    5. list_missing       — τελετές με ελλείψεις
    6. draft_supplier_message — ΠΡΟΧΕΙΡΟ μήνυμα παραγγελίας (μόνο κείμενο)

Πλεονεκτήματα:
- Πιο ακριβείς απαντήσεις: συνδυάζει πολλές συνθήκες σε μία ερώτηση.
- Λιγότερα tokens: στέλνει στο cloud μόνο όσα ζητά κάθε εργαλείο, όχι όλα.
- Λιγότερα προσωπικά δεδομένα προς το cloud (privacy).
- Συνδυάζει πολλαπλά εργαλεία στη σειρά (έως 6 βήματα).

Σειρά απάντησης (καμία αλλαγή στον client):
  1) Gemini Agent (με εργαλεία)
  2) Απλό Gemini (αν αποτύχει ο agent loop)
  3) OpenAI (αν υπάρχει key, όπως πριν)
  4) Τοπική λογική (localFallback)

Χρυσοί κανόνες (διατηρούνται):
- Όλα τα εργαλεία READ-ONLY: διαβάζουν, δεν αλλάζουν/δεν στέλνουν τίποτα.
- Το draft_supplier_message ΜΟΝΟ ετοιμάζει κείμενο για χειροκίνητη αποστολή.
- Κανένα νέο Supabase query: τα εργαλεία δουλεύουν πάνω στο payload που ήρθε.
- Ίδιο input payload, ίδια απάντηση { ok, answer }.
- Καμία αλλαγή σε app.js / index.html / styles.css.
- Καμία κλήση μοντέλου δεν ρίχνει 500: σε αποτυχία απαντά ο τοπικός βοηθός.

Ενεργοποίηση: ίδια με V41.0
- supabase secrets set GEMINI_API_KEY=...
- supabase functions deploy ai-assistant
(Το ίδιο deploy ενεργοποιεί ταυτόχρονα V41.0 + V41.2.)
